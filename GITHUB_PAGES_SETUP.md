# GitHub Pages Setup Guide for VLESS Configs

## Quick Start (Single Repository Setup)

This guide explains how to set up **one repository** that serves both as config storage AND GitHub Pages hosting.

### Step 1: Create the Repository

1. Go to https://github.com/new
2. Repository name: `vless-configs` (or your preferred name)
3. **Visibility: Public** (required for free GitHub Pages)
4. ✅ Initialize with README (optional)
5. Click **Create repository**

### Step 2: Enable GitHub Pages

1. In your new repository, go to **Settings** tab
2. Scroll down to **Pages** section (left sidebar)
3. Under **Build and deployment**:
   - Source: `Deploy from a branch`
   - Branch: Select `main` 
   - Folder: `/ (root)`
4. Click **Save**

Wait 1-2 minutes. Your Pages URL will be:
```
https://YOUR_USERNAME.github.io/vless-configs/
```

### Step 3: Configure Your Bot

In your bot's `.env` file:

```bash
# Both variables point to the SAME repository
CONFIG_REPO=your-username/vless-configs
PAGES_REPO=your-username/vless-configs
```

### Step 4: Verify Setup

After running the bot once, check:

1. **Repository contents:**
   - `configs/` directory should exist
   - Config files like `Weleredz_20240126_143022.txt`
   - `index.html` auto-generated

2. **GitHub Pages:**
   - Visit: `https://your-username.github.io/vless-configs/`
   - You should see the config index page

3. **Direct config access:**
   - `https://your-username.github.io/vless-configs/configs/Weleredz_20240126_143022.txt`

## How It Works

### When User Clicks "Get Proxy":

1. **Bot generates VLESS config** with unique UUID
2. **Uploads to GitHub:**
   - Path: `configs/{username}_{timestamp}.txt`
   - Creates `configs/.gitkeep` if directory doesn't exist
3. **Updates index.html:**
   - Scans all files in `configs/`
   - Generates new HTML table
   - Commits to repository root
4. **GitHub Pages auto-deploys** (usually < 1 minute)
5. **Config accessible** via HTTPS URL

### When User Clicks "Update":

1. **Bot preserves UUID** from existing config
2. **Updates same file** (or creates new timestamped file)
3. **Refreshes index.html**
4. **New config deployed** to Pages

## Configuration Examples

### Basic .env Setup

```bash
# Bot token from @BotFather
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz

# GitHub token with repo scope
GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxx

# Single repository for both storage and pages
CONFIG_REPO=Weleredz/vless-configs
PAGES_REPO=Weleredz/vless-configs

# Your VLESS server details
VLESS_ADDRESS=your-domain.com
VLESS_PORT=443
VLESS_PATH=/vless
VLESS_SNI=your-domain.com
```

### Custom Domain (Optional)

If you want to use your own domain:

1. Add CNAME file to repository root:
   ```
   proxy.yourdomain.com
   ```

2. Configure DNS:
   ```
   CNAME proxy.yourdomain.com username.github.io
   ```

3. Update bot config:
   ```bash
   VLESS_SNI=proxy.yourdomain.com
   ```

## Troubleshooting

### ❌ GitHub Pages Shows 404

**Solution:**
1. Wait 2-3 minutes after first upload
2. Check Settings → Pages is enabled
3. Verify `index.html` exists in root
4. Check Actions tab for deployment status

### ❌ Config Upload Fails

**Solution:**
1. Verify GITHUB_TOKEN has `repo` scope
2. Check CONFIG_REPO format: `owner/repo`
3. Ensure repository is accessible

### ❌ configs/ Directory Not Created

**Solution:**
The bot automatically creates it on first upload. If not:
1. Manually create `configs/.gitkeep` in repository
2. Run bot again

### ⚠️ Repository Must Be Public

Free GitHub Pages requires public repositories. If you need private:
- Upgrade to GitHub Pro
- Or use alternative hosting (Cloudflare Pages, Netlify, etc.)

## Security Notes

⚠️ **Important Considerations:**

1. **Public Access:** Anyone with the URL can download configs
2. **Share Carefully:** Only give config URLs to trusted users
3. **Regular Cleanup:** Delete old/unused configs
4. **Monitor Usage:** Check GitHub traffic analytics

### Best Practices:

- Use long, random filenames (bot does this automatically)
- Don't share index page URL publicly
- Rotate configs periodically
- Monitor repository access logs

## Advanced Features

### Manual Config Management

You can manually manage configs via GitHub UI:

1. **Add config:**
   - Upload file to `configs/` folder
   - Bot will include it in next index update

2. **Delete config:**
   - Remove file from `configs/`
   - Index updates automatically on next bot action

3. **Edit index.html:**
   - Customize HTML template in `core/github_manager.py`
   - Function: `_generate_index_html()`

### Multiple Users

By default, only @Weleredz can use the bot. To allow more:

Edit `bot/main.py`:
```python
ALLOWED_USERNAMES = ["Weleredz", "user2", "user3"]

def is_allowed_user(self, username: str) -> bool:
    return username in ALLOWED_USERNAMES
```

## API Reference

### Direct URLs

Once configured, configs are accessible at:

| Type | URL Format | Example |
|------|-----------|---------|
| Index Page | `https://{owner}.github.io/{repo}/` | `https://user.github.io/vless-configs/` |
| Config File | `https://{owner}.github.io/{repo}/configs/{file}` | `.../configs/user_time.txt` |
| Raw File | `https://raw.githubusercontent.com/{owner}/{repo}/main/configs/{file}` | Direct raw content |

### GitHub API Endpoints

The bot uses these GitHub API operations:

- `GET /repos/{owner}/{repo}/contents/configs` - List configs
- `PUT /repos/{owner}/{repo}/contents/{path}` - Upload/update file
- `DELETE /repos/{owner}/{repo}/contents/{path}` - Delete file
- `GET /repos/{owner}/{repo}/pages` - Get Pages info

## Maintenance

### Regular Tasks

1. **Weekly:** Review config files, delete old ones
2. **Monthly:** Check GitHub Pages bandwidth usage
3. **Quarterly:** Rotate server credentials if needed

### Cleanup Script

Example to remove configs older than 30 days:

```bash
#!/bin/bash
cd /path/to/repo
find configs/ -name "*.txt" -mtime +30 -exec rm {} \;
git add -A
git commit -m "Cleanup old configs"
git push
```
